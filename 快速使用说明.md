# 快速使用说明

前提：对方已经安装并能正常使用 OpenClaw。

## 当前推荐方式

优先不要只依赖导入脚本。

正确交付方式是：

1. 把这个仓库拉到客户本地
2. 在仓库目录打开 Codex
3. 让客户本地 Codex 按 `CUSTOMER_LOCAL_IMPORT.md` 执行本地导入

这样 Codex 会直接依据仓库里的 Agent 架构和详细配置，把内容导入到客户自己的 OpenClaw 环境，并先处理备份、冲突检查、配置合并和校验。

## 辅助脚本方式

### Windows

```powershell
git clone https://github.com/pengke531/openclaw-creator-outreach-opc-kit
cd openclaw-creator-outreach-opc-kit
powershell -ExecutionPolicy Bypass -File .\import.ps1
```

### macOS / Linux / WSL2

```bash
git clone https://github.com/pengke531/openclaw-creator-outreach-opc-kit
cd openclaw-creator-outreach-opc-kit
chmod +x ./import.sh
./import.sh
```

## 导入成功后

会得到：

- `main` = 旺财
- `laicai`
- `facai`

以及：

- `workspace/registry`
- `workspace/evidence`
- `workspace/inbox`

## 如果 gateway 还在启动

等 gateway 完全启动后再执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\verify-creator-outreach.ps1
```
